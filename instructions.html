Targets

You need to have the targets package installed:

install.packages("targets")
library(targets)
install.packages("tarchetypes")

Have a look at the _targets.R file and try to understand what will happen when you execute the pipeline for the first time.

Then do just that:

targets::tar_make()

You have now (hopefully) downloaded a zip file with all data needed.

Lets unzip the zip file for easier access to the individual files it contains. Using targets, this should be performed by specifying a target in the _targets.R file.

The already defined target provide you with the filename to the zip-file (what is it?) in the object zipdata.

We therefore use zipdata as argument to the the unzip-function. Afer unzipping the folder, by zip::unzip() we also return the file names of all files put into the newly created data-fixed folder.

  1. Copy and paste the following line to the relevant place in the _targets.R file. Save the file (Ctrl/Cmd + s)
  2. Run tar_make() interactively from the console to execute the now updated pipeline.
tar_target(csv_files, zip::unzip(zipdata))

All data files (.csv) should now be found in the data-fixed folder (make sure that is the case).

Data

The data previously unpacked is synthetic data from the Synthea project.

ImportantSynthetic data

Note that no personal details are “real”! The data are all fake, alhough it aims to be realistic. As such, we can use it for learning purposes and we do not need to be afraid of sharing the data publically or transfering it between computers etc. Note, however, that the data can not be used for any “real” discovery of scientific importance! Hence, you might discover some interesting patterns or associations in the data which might be fictitious!

fread

We need to read in the csv data before we can work with it in R. We like data.table objects when working with big data and can use data.table::fread() for individual csv files.

We could also take all the csv file names from the data-fixed folder and make a big list of data.table objects. Try what happens if you run (and read the necessary function documentation to understand what you do) this code:

When working within a pipeline it can be handy, however, to convert all csv data sets and cache them using a more efficient file format. We have instructed targets to use the qs2 storage format internally (its on one of the first lines in _targets.R although it is there called qs and not qs2; see [qs2](https://github.com/qsbase/qs2)).

In this case, however, we can use targets to not only read the files but also to store individual qs2-files for each original csv file. The code is a bit complex so it’s already inserted in the pipeline.

Labels

Codebooks/variable list are somewhat helpful but they usually exist as a separate document (a PDF, or web page as illustrated above). We showed how to convert the documentation to something more useful above but wouldn’t it be nice to integrate it more directly with the data? People working with SPSS data are quite used to having labels attached to the column names. This is not the default in R but the package labelled (https://larmarange.github.io/labelled/) can be used (install it!).

Add labels to the patient data:

labelled::var_label(dt_patients) <-
  cb_patients[, setNames(description, column_name)]

Now, what does ssn actually mean?

labelled::look_for(dt_patients, "ssn")

What does income mean?

labelled::look_for(dt_patients, "income")

Well that’s some level of detail but is it enough for analysis? Does this include all sorts of income? Salary, insurance, benefits, pensions etc? What is the unit of measurement? SEK? DKK? USD? Does the value need to be multiplied by 1000 to get the actual value? Is it income from the last known year, an averager over time or from some other year?

dt_patients[, hist(income)]

Woo, we have some rich people here! Who are they? Where can I find them? Let me check their adresses … WAIT, WHAT? Does our ethical permission allow such use of the data?

Keys

The data originates from a database where the concept of primary and foreign keys exists. This could be seen by the key column from the codebook. The primary key is a unique identifier for each row in a dataset. Which ones do we have?

tar_read(codebook)[key == "primary", .(file, column_name)]

Let’s set the key for each of the datasets where this is relevant.

Add the id key to dt_patients by:

setkey(dt_patients, id)

We could do this by adding new targets to the pipeline. If so, however, we would end up with multiple versions of each cached data set. It seems unnecessary. Let’s instead go back and add the keys aready before they are cached the first time. You were previsously asked to uncomment a line including the fread() function. We could add the key argument here to use the id column as key. The problem is that not all data sets have such id columns so you would need something. Try to found a solution if you have time and interest (otherwise not).
